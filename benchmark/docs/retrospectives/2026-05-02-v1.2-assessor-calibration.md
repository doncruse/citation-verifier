# 2026-05-02 — v1.1 assessor calibration

> *Originally scoped as v1.2 in `benchmark-roadmap.md`. Rebadged to v1.1 because it's a small additive validation study, not a separate release line. The filename keeps the original `v1.2` slug for git stability.*

**Goal:** test whether Sonnet 4.6 or Haiku 4.5 could substitute for Opus 4.7 as the substance assessor on v1's deduped 257-cell subset. If yes, N=500–1000 becomes affordable within Opus's effective budget; if no, v1.x stays at N≈200.

**Outcome:** both fail the bar. Stick with Opus.

## Headline

| Model | Overall agreement | Red recall | Red precision | Cohen's κ |
|---|---:|---:|---:|---:|
| Sonnet 4.6 | 68.9% (177/257) | 52.2% | 87.8% | 0.50 |
| Haiku 4.5 | 65.4% (168/257) | 55.1% | 67.9% | 0.41 |
| *Bar (per `benchmark-roadmap.md` v1.1, originally v1.2)* | ≥90% | ≥85% | — | — |

Both miss by ~20pp on overall and ~30pp on Red recall — not borderline. Decision is robust under the n=257 CIs (≈±5.6pp on overall, ≈±12pp on Red recall).

Detail lives in [`benchmark/releases/v1/calibration.md`](../../benchmark/releases/v1/calibration.md) (confusion matrices, per-class precision/recall, costs).

## What worked

- **Direct Anthropic API at `temperature=0`** kept the noise floor near zero and removed the "did the model just roll the dice differently" objection that would have shadowed CLI runs.
- **Reusing v1's `ASSESSMENT_PROMPT` verbatim** kept the question scoped to "drop-in replacement" rather than "drop-in with prompt re-tuning."
- **Reusing the cached opinion text from `benchmark/pilot_a/cited_opinion_cache`** — zero CourtListener calls during calibration; whole study was 514 Anthropic calls and ~20 minutes of wall clock.
- **Dedup-before-calibration**: filtering to the canonical 257 cells (via `scorecard.canonical_dataset_ids()`) prevented over-weighting the 35% of duplicate propositions in the raw 600.
- **Resume-safe append-only output CSV** kept the run robust to interruption.

## What didn't

### Keying bug (caught after the run completed)

First-run resume key was `(id, candidate_model)`, but `id` is shared across the up-to-3 v1 cells per example (one per model-under-test, sonnet/opus/gpt-5). The script visited all such cells per id but the resume `seen` set didn't refresh mid-loop, so it processed every cell — except the output CSV didn't have a `model_under_test` column to record which cell each call corresponded to.

Result: 502 of 514 expected calls landed in the CSV, ambiguous between cells.

**Recovery:** salvageable. The script visited cells in `results.csv` row order, and `opus_verdict` on each calibration row aligned exactly with the `supports` column of the v1 cell at the same index — zero mismatches across all 234 fully-aligned pairs. Migration script ([`benchmark/runners/_migrate_calibration_csv.py`](../../benchmark/runners/_migrate_calibration_csv.py)) re-keyed via order-based join, then a 12-call fill-in run completed the 6 triples that the smoke run had partially blocked. No data lost.

**Fix in the script:** keying changed to `(id, model_under_test, candidate_model)`; output CSV now records `model_under_test`. Backup of the broken-keying CSV preserved as `calibration_results_original.csv` so the postmortem is reproducible.

### Wrong-cache smoke

First smoke pointed at `benchmark/releases/v1/citing_opinion_cache` (citing-court opinions used during dataset mining) instead of `benchmark/pilot_a/cited_opinion_cache` (cited-case opinions, which is what the assessor actually needs). All 3 smoke cells returned "no opinion text," wasting one round-trip. Fixed in code with an inline comment so the next person doesn't repeat it.

### `python-dotenv` doesn't override pre-existing env vars

`ANTHROPIC_API_KEY` was set as an empty string in the system environment on Windows (probably from a stale shell profile). `load_dotenv(.env)` silently kept the empty pre-existing value. Fix: `load_dotenv(.env, override=True)`.

## What this closes off

- The "5–10× the run with Sonnet" cost path is dead. N=500–1000 within reasonable Opus budget is not unlocked.
- v1.1 keeps Opus 4.7 as the primary assessor.

## What this opens up

The next-step decision shifts to v1.1 methodology fix-ups that don't depend on a cheaper assessor:

- Mining-stage dedup (the bug that demoted N=200 → 130)
- Per-case metadata extraction + stratified sampling by tier (enables district-vs-appellate hallucination-rate analysis)
- Verified-citations cache (speed/reliability)

Within the existing Opus-budget envelope, these get a stronger v1.1 that retires the dedup footnote and supports tier-breakdown analysis.

## Cost & time

- $5.43 in Anthropic API spend ($4.06 Sonnet + $1.37 Haiku) for 514 calls
- ~20 minutes wall clock for the full Sonnet+Haiku run (Sonnet ~12 min, Haiku ~7 min)

## Future work explicitly NOT done here

- **Prompt re-tuning for cheaper models.** This study tested drop-in replacement under the v1 prompt. A separate study could ask "can Sonnet/Haiku assess with a prompt tuned for them?" — different question, different budget, separate scope.
- **Re-Opus noise floor.** Considered and dropped: temp=0 collapses the noise floor to ~1%, so the bar gap (~20pp) dwarfs any plausible noise correction.
